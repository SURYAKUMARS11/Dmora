1. Prepare Your Accounts

To automate Instagram with the Meta Graph API, you must link your Instagram page to a Facebook Page:

  • Convert your Instagram account to a Professional Account (either Creator or Business).
  • Create a Facebook Page for your business or brand.
  • In Instagram App, go to Settings → Creator Tools → Connect/Link a Page, and select your FB Page. Alternatively, link it via Meta Business Suite.
  • Crucial Option: In Instagram, go to Settings → Privacy → Messages and make sure **"Allow Access to Messages"** is turned ON. Without this, external apps cannot read or send DMs.

2. Create a Meta Developer Application

Set up an application on the Meta Developer Portal to access the Instagram endpoints:

  • Go to the Meta Developer Portal and log in.
  • Click My Apps → Create App.
  • Select Other for Use Case, then select Business as App Type. Give it a name.
  • Under "Add products to your app", find Instagram Graph API and click Set Up.
  • Add Messenger product and click Set Up (needed for sending direct messages).

3. Obtain Page Access Token & Configure Webhooks

Link your Meta App to your Facebook Page to generate the required credentials:

  • In the App Dashboard sidebar, go to Messenger → Instagram Settings.
  • Under Access Tokens, link your Facebook Page. Click **Generate Token**.
  • Copy this generated token and paste it into the Page Access Token input in the Bot Settings tab.
  • Under Webhooks in the app dashboard, click **Configure**.
    • Callback URL: your public HTTPS endpoint (e.g. `https://xxxx.ngrok-free.app/webhook`)
    • Verify Token: must match the Webhook Verify Token configured in your settings panel.
  • Once verified, look under **Webhook fields** and click **Subscribe** next to comments.

4. Public Webhook Tunneling (ngrok)

Since Meta needs to send webhook requests to a public `https` URL, you need to expose your local server using a tunneling service like ngrok during development:

  1. Download and install ngrok.
  2. Start your local Express server (port `3000` by default): 
    npm run dev
  3. In a separate terminal, expose port 3000 using ngrok: 
    ngrok http 3000
  4. ngrok will generate a forwarding URL like:
    https://a4b1-80-120-40-5.ngrok-free.app
  5. Copy that HTTPS URL, append /webhook, and use it as your Callback URL in Meta's Webhook Settings. E.g.:
    https://a4b1-80-120-40-5.ngrok-free.app/webhook
Ready to Deploy?

For production, deploy this server to cloud services like Render, Fly.io, or Heroku, configure your environment variables, and use their public HTTPS endpoint as the permanent webhook URL!
